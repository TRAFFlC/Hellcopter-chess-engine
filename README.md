# Chess Engine Bot for 国象联盟

一个面向 [国象联盟](https://www.chessleague.cn/) 平台的自动对弈国际象棋引擎工具。通过屏幕截图识别棋盘、引擎计算走法、模拟鼠标点击执行走棋，实现全自动对弈。

## 引擎定位

- **计算资源**：纯 CPU 计算，无需 GPU
- **时间控制**：Bullet（子弹棋），当前默认 5+2（5 分钟 + 2 秒加秒）
- **预估等级**：国象联盟 ELO 约 **1900–2200**
- **搜索性能**：NPS（每秒节点数）约 300K+，2 秒内可达 8–10 层搜索深度

## 工作原理

### 整体架构

```
game.py (游戏主循环)
  → board.py (屏幕截图 + 图像差分识别对手走法)
  → engine.py (Python 引擎包装器)
    → engine_wrapper.py (ctypes 接口层)
      → engine_core.dll (C 语言核心搜索引擎)
  → screen.py (模拟鼠标点击执行走法)
```

项目采用 **Python + C 混合架构**：Python 负责 UI 自动化与视觉识别，C 负责高性能搜索与评估，通过 ctypes 桥接调用。

### 与国象联盟的交互方式

本项目**不依赖任何 API 或网络协议**，通过计算机视觉与 UI 自动化方式与国象联盟交互：

1. **棋盘定位**：使用 OpenCV 模板匹配识别开局棋盘图片，确定棋盘位置和己方颜色（白/黑）
2. **走法检测**：通过前后两帧截图的图像差分（`cv2.absdiff`）检测对手走法，将像素变化映射为合法走法
3. **走法执行**：计算目标格子的屏幕坐标，用 `pyautogui.click()` 模拟鼠标点击
4. **升变处理**：检测升变选择界面，自动点击后图标
5. **反检测**：走法间随机延迟 0.5–5.5 秒，模拟人类思考时间

### 搜索算法（C 引擎核心）

C 引擎实现了完整的现代国际象棋搜索框架：

| 技术 | 说明 |
|------|------|
| **迭代加深** | 从深度 1 逐层加深，每层将最佳走法移至首位用于下一层排序 |
| **Negamax + Alpha-Beta** | 基础搜索框架，带 Alpha-Beta 剪枝 |
| **PVS（主变例搜索）** | 第一个走法全窗口搜索，后续走法零窗口验证，失败再重搜 |
| **静止搜索** | 搜索吃子/升变/将军着法至局面稳定，避免水平线效应 |
| **空着裁剪** | 深度 ≥ 4 时执行，R=3，深度 ≥ 6 增加验证搜索 |
| **将军延伸** | 被将军时深度 +1（最多延伸 4 次） |
| **置换表** | 2^20 条目，深度优先替换，存储 EXACT/UPPER/LOWER 三种类型 |
| **着法排序** | 置换表最佳着法 > MVV-LVA 吃子 > 杀手着法 > 历史启发 |
| **重复检测** | 搜索历史 + 游戏历史双重检测，和棋返回 0 分 |

### 评估函数

评估函数约 730 行 C 代码，覆盖十余个维度：

- **材料价值 + PST 插值**：所有棋子均有中局/残局双表 PST，通过游戏阶段（phase）线性插值
- **兵结构**：叠兵惩罚、孤兵惩罚、通路兵奖励、兵链奖励
- **双象奖励**：同时拥有双象 +55
- **车评估**：开放线/半开放线奖励、第七横线车奖励
- **象活动性**：好象/坏象评估
- **王安全**：王周围攻击权重、兵盾奖励、开放线威胁、底线杀王威胁
- **马评估**：前哨站奖励、边缘马惩罚
- **子力被攻击/保护**：被兵攻击惩罚、后被轻子攻击惩罚、马叉击惩罚
- **中心控制**：中心/扩展中心占子/兵奖励
- **开局发展**：未出子惩罚、王未易位惩罚
- **子力不平衡**：多轻子少兵惩罚，防止不合理弃子

## 快速开始

### 环境要求

- Python 3.10+
- C 编译器（GCC / MSVC / Clang）
- Windows 操作系统（依赖 `pyautogui` 屏幕操作）
- 国象联盟桌面客户端

### 安装

```bash
git clone https://github.com/<your-username>/chess.git
cd chess
pip install -r requirements.txt
python build_engine.py
```

### 运行

**GUI 模式（推荐）**：

```bash
python main.py
```

启动 Tkinter 控制面板，可调整引擎深度和时间限制，点击开始后自动对弈。

**人机对弈**：

```bash
python play_gui.py
```

在本地 GUI 中与引擎对弈，无需国象联盟客户端。

### 引擎参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `ENGINE_MAX_DEPTH` | 8 | 最大搜索深度 |
| `ENGINE_TIME_LIMIT` | 2.0 | 每步时间限制（秒） |

可在 GUI 中实时调整，或修改 [config.py](config.py)。

## 项目结构

```
├── main.py              # 程序入口（CLI / GUI 模式）
├── engine.py            # Python 引擎包装器
├── engine_core.c        # C 语言核心搜索引擎
├── engine_core.h        # C 引擎头文件
├── engine_wrapper.py    # Python-C ctypes 接口层
├── build_engine.py      # C 引擎编译脚本
├── config.py            # 全局配置常量
├── game.py              # 游戏主循环（自动对弈）
├── board.py             # 棋盘视觉识别
├── screen.py            # 屏幕截图与鼠标操作
├── gui.py               # Bot 控制面板 GUI
├── play_gui.py          # 人机对弈 GUI
├── benchmark_engine.py  # 引擎性能基准测试
├── requirements.txt     # Python 依赖
└── *.png                # UI 模板匹配图片
```

## 技术栈

- **搜索引擎**：C（Negamax + PVS + 静止搜索 + 置换表）
- **视觉识别**：OpenCV（模板匹配 + 图像差分）
- **UI 自动化**：pyautogui（截图 + 鼠标模拟）
- **棋局逻辑**：python-chess（FEN 解析 + 合法走法生成）
- **GUI**：Tkinter
